\# Design Direction



\## Core Design Goal

Use the uploaded HTML file as the primary visual direction reference for this MVP.



Important:

\- inherit the overall tone, rhythm, palette, spacing logic, and UI language

\- do NOT reuse the logo, icons, placeholder images, or brand-specific graphics from the HTML

\- do NOT copy visual assets literally

\- translate the style into a cleaner production-ready system



The uploaded HTML establishes the key palette and UI language through dark backgrounds, teal/green primary accents, lighter accent highlights, bold typography, mono micro-labels, soft cards, and fixed navigation. :contentReference\[oaicite:21]{index=21} :contentReference\[oaicite:22]{index=22}

\## Visual Reference Rules



\### Keep

\- dark background system

\- teal / green accent logic

\- large typographic hierarchy

\- mono uppercase section labels

\- card-based information layout

\- soft borders and subtle glass / overlay feel

\- strong spacing and breathable sections

\- minimal visual clutter



\### Do Not Keep

\- existing logo

\- lightning icon

\- emoji placeholders

\- current illustrative placeholders

\- current decorative visuals as final content



The reference HTML uses a fixed, blurred dark nav bar, accent-led buttons, hero gradients, and modular section blocks; these are the right direction for the new site. :contentReference\[oaicite:23]{index=23} :contentReference\[oaicite:24]{index=24}

\## Color Direction

Use the HTML palette as the starting point.



Suggested system:

\- Primary: deep teal

\- Accent: bright green-teal

\- Background: deep blue-black / charcoal

\- Surface: muted blue-green dark panels

\- Text: soft off-white

\- Secondary text: desaturated blue-gray



Approximate palette reference from the uploaded HTML:

\- `--clr-primary: #0A5E5C`

\- `--clr-accent: #2DD4A8`

\- `--clr-dark: #0B1A1F`

\- `--clr-surface: #163038`

\- `--clr-text: #E8F4F0`

\- `--clr-text-dim: #8AACB8`



These values come directly from the HTML reference and should guide the design system. :contentReference\[oaicite:25]{index=25}



---



\## Typography

Direction:

\- bold, high-clarity headline system

\- clean sans-serif for body

\- mono style for section labels / metadata



Reference behavior:

\- large clamp-based headlines

\- uppercase mono labels with letter spacing

\- strong line-height and high readability



This reflects the uploaded HTML's title, subtitle, label, and hero typography patterns. :contentReference\[oaicite:26]{index=26} :contentReference\[oaicite:27]{index=27}



---



\## Layout

\- wide desktop container

\- strong section spacing

\- modular card grids

\- simple page rhythm

\- no heavy imagery required in MVP

\- prioritize readability and hierarchy over animation



Suggested section rhythm:

\- generous top/bottom padding

\- consistent headline widths

\- easy scanning on desktop

\- clear stacking on mobile



---



\## Homepage Priority

Homepage should visually emphasize:

1\. positioning

2\. scenarios

3\. solutions

4\. platform trust

5\. products

6\. safety

7\. service / Japan support



Do not let Products dominate too early.



---



\## Product Presentation

Products should use:

\- actual product information from the source PDF

\- actual product images captured from the source PDF where useful

\- consistent product-card structure

\- expandable family sections



Avoid turning the website into a dense datasheet catalog.



---



\## Motion

Keep motion subtle:

\- fade-in on scroll

\- light hover states

\- no heavy animation

\- no background videos for current MVP



The current reference HTML already uses restrained motion behavior and section reveals; that level is appropriate. :contentReference\[oaicite:28]{index=28}



---



\## Responsive Behavior

The uploaded HTML already demonstrates a mobile collapse pattern and simplified responsive grids; retain this responsive philosophy while rebuilding in a cleaner component system. :contentReference\[oaicite:29]{index=29}



Requirements:

\- fully responsive

\- mobile navigation

\- clear typography on small screens

\- no broken cards

\- no image-heavy dependencies
